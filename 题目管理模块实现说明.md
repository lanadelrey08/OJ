# OJ 题目管理模块实现说明

## 1. 模块范围

实现题目管理模块，覆盖以下能力：

- `GET /api/v1/problems`
- `GET /api/v1/problems/{id}`
- `POST /api/v1/problems`
- `PUT /api/v1/problems/{id}`
- `DELETE /api/v1/problems/{id}`

## 2. 工程结构

```text
src/main/java/com/oj/problem
├── common
├── controller
├── dto
├── entity
├── exception
├── repository
├── security
└── service
```

## 3. 模块说明

- Controller 负责接口层、参数校验和鉴权入口
- Service 负责题目、标签、测试用例的一致性维护
- Repository 负责访问 `problems`、`test_cases`、`tags`、`problem_tags`
- 鉴权基于 JWT，要求 `Authorization: Bearer <JWT_TOKEN>`
- 管理员接口要求 `role=admin`

JWT payload 可采用：

```json
{
  "user_id": 1,
  "role": "admin"
}
```

## 4. 已实现能力

- 题目分页查询
- 难度、标签、关键词筛选
- 公开题目详情查询
- 管理员创建题目
- 管理员更新题目
- 管理员删除题目
- 题目标签自动增删和计数维护
- 测试用例随题目统一维护
- 公开题目详情不暴露完整测试用例
- 统一响应体和错误码返回

## 5. 数据库脚本

初始化 SQL 位于 `src/main/resources/schema.sql`，包含：

- `users`
- `problems`
- `test_cases`
- `tags`
- `problem_tags`

## 6. 运行方式

修改 `src/main/resources/application.yml` 中的数据库和 JWT 配置后，可用 Maven 启动：

```bash
mvn spring-boot:run
```

或打包运行：

```bash
mvn clean package
java -jar target/problem-management-1.0.0.jar
```
