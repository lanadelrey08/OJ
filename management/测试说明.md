# 测试说明

## 1. 已添加内容

- JUnit 5 测试
- Mockito 单元测试
- MockMvc 控制器测试
- JaCoCo 覆盖率统计与校验
- H2 测试数据库配置

## 2. 覆盖范围

测试重点覆盖以下有业务价值的代码：

- `ProblemServiceImpl`
- `ProblemController`
- `JwtTokenService`
- `GlobalExceptionHandler`
- `ApiResponse`

JaCoCo 已配置 80% 的行覆盖率校验门槛。

## 3. 如何运行

### 方式一：命令行

```bash
mvn clean verify
```

执行完成后可查看：

- `target/site/jacoco/index.html`

### 方式二：IDEA

1. 将项目作为 Maven 项目导入
2. 确认 JDK 使用 8 或更高版本
3. 等待 Maven 依赖下载完成
4. 运行 `src/test/java` 下的测试
5. 如需看覆盖率，可直接运行 `verify` 或用 IDEA 的 Coverage 功能

## 4. 环境要求

- JDK 8+
- Maven 3.8+
- 能访问 Maven Central 下载依赖

## 6. 当前项目中的测试配置

- 测试数据库配置文件：`src/test/resources/application-test.yml`
- 覆盖率插件配置：`pom.xml`
- 若只跑单元测试，不依赖真实 MySQL

## 7. 说明

为了让覆盖率更准确反映业务逻辑，JaCoCo 排除了纯数据载体和样板类：

- `dto`
- `entity`
- `repository`
- 启动类
