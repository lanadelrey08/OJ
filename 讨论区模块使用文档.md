# 讨论区模块使用文档

## 1. 模块结构

讨论区模块位于 `OJ/discussion` 目录下，采用模块化设计，包含以下核心组件：

```
discussion/
├── include/                    # 头文件目录
│   ├── discussion_handler.h    # HTTP接口处理器定义
│   └── discussion_service.h    # 讨论区业务服务定义
├── src/                        # 源文件目录
│   ├── discussion_handler.cpp  # HTTP接口实现
│   ├── discussion_service.cpp  # 业务逻辑实现
│   └── main.cpp                # 主程序入口
├── tests/                      # 测试文件目录
│   └── test_discussion_service.cpp  # 业务服务单元测试
└── CMakeLists.txt              # CMake配置文件
```

## 2. 核心组件

### 2.1 讨论服务（DiscussionService）

负责讨论区核心业务逻辑，当前版本采用线程安全内存存储，支持主题与评论管理。

**主要结构体**：
- `DiscussionTopic`：讨论主题结构（主题ID、题目ID、用户ID、标题、内容、时间、评论数）
- `DiscussionComment`：评论结构（评论ID、主题ID、用户ID、父评论ID、内容、时间）

**主要类**：
- `DiscussionService`：讨论区业务服务类

**主要能力**：
- 创建主题 `createTopic`
- 查询主题列表 `listTopics`（支持按 `problem_id` 过滤与分页）
- 查询主题详情 `getTopic`
- 创建评论 `createComment`（支持回复评论）
- 查询主题评论列表 `listComments`

**使用方法**：
```cpp
#include "discussion_service.h"
#include <optional>

DiscussionService service;

int64_t topic_id = service.createTopic(
    1001,
    42,
    "Two Sum 求解思路",
    "谁能解释 O(n) 解法？");

int64_t comment_id = service.createComment(
    topic_id,
    7,
    "建议使用哈希表。",
    std::nullopt);

auto topics = service.listTopics(std::nullopt, 20, 0);
auto comments = service.listComments(topic_id);
```

### 2.2 接口处理器（DiscussionHandler）

负责 HTTP 路由注册、请求参数校验、调用业务层并返回 JSON 响应。

**主要类**：
- `DiscussionHandler`：讨论区接口处理器类

**功能说明**：
- 参数校验（必填字段、长度限制、分页参数）
- JSON 响应构建
- 统一错误处理（使用 `oj::HttpException` 与 `oj::makeErrorJson`）
- 跨域配置（CORS）

### 2.3 服务入口（main）

`main.cpp` 负责讨论区服务启动流程：
- 读取公共配置 `oj::loadConfigFromEnv`
- 初始化公共基础设施 `oj::initInfrastructure`
- 启动 `DiscussionHandler` HTTP 服务
- 进程退出时执行 `oj::shutdownInfrastructure`

## 3. 配置说明

### 3.1 讨论区模块环境变量

| 变量 | 说明 | 默认 |
|------|------|------|
| `OJ_DISCUSSION_PORT` | 讨论区服务监听端口 | 8090 |

### 3.2 公共服务环境变量

讨论区模块复用 `common` 公共基础设施（日志、MySQL连接池、Redis连接）。
可选环境变量包括：

- `OJ_MYSQL_HOST` / `OJ_MYSQL_PORT` / `OJ_MYSQL_USER` / `OJ_MYSQL_PASSWORD` / `OJ_MYSQL_DATABASE`
- `OJ_REDIS_HOST` / `OJ_REDIS_PORT` / `OJ_REDIS_PASSWORD` / `OJ_REDIS_DB`
- `OJ_LOG_FILE` / `OJ_LOG_LEVEL`

说明：当前讨论区业务数据为内存实现，不依赖 MySQL/Redis 进行主题与评论持久化；`/health` 接口会返回公共基础设施的就绪状态。

## 4. 构建与运行

### 4.1 构建项目（MinGW Makefiles）

在 OJ 根目录执行：

```powershell
# 创建构建目录
New-Item -ItemType Directory -Force -Path build_mingw
cd build_mingw

# 配置项目
cmake .. -G "MinGW Makefiles"

# 构建讨论区模块（服务+测试）
cmake --build . --target discussion_server test_discussion_service
```

### 4.2 运行测试

```powershell
# 方式1：直接运行测试程序
./discussion/test_discussion_service.exe

# 方式2：使用 CTest
ctest --test-dir discussion --output-on-failure
```

### 4.3 运行讨论区服务

```powershell
# 可选：指定端口
$env:OJ_DISCUSSION_PORT="8090"

# 启动服务
./discussion/discussion_server.exe
```

## 5. HTTP接口说明

### 5.1 创建主题

- URL: `/api/discussions/topics`
- Method: `POST`
- Body: JSON，必填字段
  - `problem_id` (int64)
  - `user_id` (int64)
  - `title` (string, 最大 200)
  - `content` (string, 最大 5000)

**示例请求**：
```json
{
  "problem_id": 1001,
  "user_id": 1,
  "title": "Two Sum 求解思路",
  "content": "谁能解释 O(n) 解法？"
}
```

### 5.2 查询主题列表

- URL: `/api/discussions/topics`
- Method: `GET`
- Query 参数（可选）
  - `problem_id`：按题目过滤
  - `limit`：分页大小，默认 20，最大 200
  - `offset`：偏移量，默认 0

### 5.3 查询主题详情

- URL: `/api/discussions/topics/{topic_id}`
- Method: `GET`

### 5.4 创建评论

- URL: `/api/discussions/topics/{topic_id}/comments`
- Method: `POST`
- Body: JSON，必填字段
  - `user_id` (int64)
  - `content` (string, 最大 3000)
  - `parent_comment_id` (int64, 可选，回复评论时使用)

**示例请求**：
```json
{
  "user_id": 2,
  "content": "建议使用哈希表。",
  "parent_comment_id": null
}
```

### 5.5 查询评论列表

- URL: `/api/discussions/topics/{topic_id}/comments`
- Method: `GET`

### 5.6 健康检查

- URL: `/health`
- Method: `GET`
- 返回服务状态与公共基础设施状态

## 6. 集成方法

### 6.1 作为库集成

将讨论区模块作为静态库集成到其他项目中：

```cmake
# 在 CMakeLists.txt 中添加
add_subdirectory(path/to/discussion)
target_link_libraries(your_project discussion_core)
```

### 6.2 通过 HTTP 接口集成

前端或网关可直接通过 REST API 调用讨论区服务：
- 主题创建、查询
- 评论创建、查询
- 健康检查

## 7. 注意事项

1. **数据持久化**：
   - 当前版本为内存存储，进程重启后数据会丢失。

2. **并发安全**：
   - `DiscussionService` 内部使用互斥锁保证线程安全。

3. **参数校验**：
   - 已实现字段必填与长度校验，但未实现更复杂的内容审核。

4. **权限体系**：
   - 当前接口示例以 `user_id` 字段直接传入，尚未接入完整认证鉴权链路。

5. **公共依赖**：
   - 模块依赖 `common` 与 `judge/include` 中的 Crow 头文件。

## 8. 故障排除

### 8.1 构建失败（找不到 MinGW Make）

- 检查 `mingw32-make` 是否可用：
```powershell
mingw32-make --version
```
- 确保 MinGW `bin` 目录在 PATH 中。

### 8.2 端口占用导致服务启动失败

- 修改端口后重启：
```powershell
$env:OJ_DISCUSSION_PORT="8091"
./discussion/discussion_server.exe
```

### 8.3 接口返回 `bad_request`

- 检查 JSON 是否包含必填字段。
- 检查字段类型和长度限制（主题标题 200、主题内容 5000、评论内容 3000）。

### 8.4 评论创建返回 `topic_not_found`

- 检查 `topic_id` 是否存在。
- 若使用 `parent_comment_id`，确认父评论属于同一主题。

## 9. 示例代码

### 9.1 PowerShell 接口调用示例

```powershell
# 1) 创建主题
$topicBody = @{
  problem_id = 1001
  user_id = 1
  title = "Two Sum 求解思路"
  content = "谁能解释 O(n) 解法？"
} | ConvertTo-Json

$topic = Invoke-RestMethod -Method Post `
  -Uri http://127.0.0.1:8090/api/discussions/topics `
  -ContentType "application/json" `
  -Body $topicBody

# 2) 查询主题列表
Invoke-RestMethod "http://127.0.0.1:8090/api/discussions/topics?problem_id=1001&limit=20&offset=0"

# 3) 创建评论
$commentBody = @{
  user_id = 2
  content = "建议使用哈希表。"
  parent_comment_id = $null
} | ConvertTo-Json

Invoke-RestMethod -Method Post `
  -Uri ("http://127.0.0.1:8090/api/discussions/topics/" + $topic.id + "/comments") `
  -ContentType "application/json" `
  -Body $commentBody

# 4) 查询评论列表
Invoke-RestMethod ("http://127.0.0.1:8090/api/discussions/topics/" + $topic.id + "/comments")
```

### 9.2 C++ 业务层调用示例

```cpp
#include "discussion_service.h"
#include <iostream>
#include <optional>

int main() {
    DiscussionService service;

    int64_t topic_id = service.createTopic(
        1001,
        1,
        "Two Sum 求解思路",
        "谁能解释 O(n) 解法？");

    int64_t first_comment = service.createComment(
        topic_id,
        2,
        "建议使用哈希表。",
        std::nullopt);

    service.createComment(
        topic_id,
        3,
        "同意楼上。",
        first_comment);

    auto topic = service.getTopic(topic_id);
    auto comments = service.listComments(topic_id);

    std::cout << "topic_id=" << topic->id
              << ", comment_count=" << topic->comment_count
              << ", comments_size=" << comments.size()
              << std::endl;
    return 0;
}
```

## 10. 总结

讨论区模块为 OJ 系统提供了完整的基础讨论能力，包括主题与评论管理、分页查询、回复链支持、健康检查接口与单元测试。当前实现以稳定、可扩展的模块骨架为目标，后续可平滑扩展到 MySQL 持久化、Redis 缓存、认证鉴权与内容审核能力。
