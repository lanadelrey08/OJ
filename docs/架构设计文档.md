# 在线判题系统（OJ）- 架构设计文档

## 一、系统概述

### 1.1 项目背景
在线判题系统（Online Judge，简称OJ）是一个用于编程练习和算法竞赛的在线平台。用户可以在线提交代码，系统自动编译运行并判定结果。

### 1.2 设计目标
- **高可用性**：系统能够7x24小时稳定运行
- **高并发**：支持多用户同时提交代码
- **安全性**：防止恶意代码执行，保护用户数据
- **可扩展性**：支持新增编程语言和功能扩展
- **易维护性**：模块化设计，便于后期维护

### 1.3 技术选型

| 层级 | 技术 | 说明 |
|------|------|------|
| 前端 | Vue.js 3.x | 响应式前端框架 |
| 后端 | C++17 + Crow | 高性能HTTP服务 |
| 数据库 | MySQL 8.0 | 关系型数据库 |
| 缓存 | Redis 7.0+ | 高性能缓存 |
| 消息队列 | RabbitMQ | 异步任务处理 |
| 沙箱 | Docker + 进程隔离 | 安全运行环境 |
| 部署 | Docker Compose | 容器化部署 |

### 2.2 模块划分

#### 2.2.1 用户管理模块
- **职责**：用户注册、登录、权限管理
- **核心功能**：
  - 用户注册/登录/登出
  - JWT认证
  - 用户信息管理
  - 权限控制

#### 2.2.2 题目管理模块
- **职责**：题目发布、编辑、分类管理
- **核心功能**：
  - 题目CRUD操作
  - 题目分类与标签
  - 测试用例管理
  - 题目搜索与筛选

#### 2.2.3 代码评测模块
- **职责**：代码提交、编译运行、结果判定
- **核心功能**：
  - 代码提交处理
  - 编译与运行
  - 沙箱安全隔离
  - 评测结果判定

#### 2.2.4 排行榜模块
- **职责**：用户排名、题目统计
- **核心功能**：
  - 总排行榜计算
  - 题目通过率统计
  - 用户解题记录
  - 比赛排名（可选）

#### 2.2.5 讨论区模块
- **职责**：题目讨论、评论管理
- **核心功能**：
  - 讨论主题管理
  - 评论与回复
  - 内容审核
  - 点赞与排序

#### 2.2.6 公共服务模块
- **职责**：基础设施与通用服务
- **核心功能**：
  - HTTP服务器框架
  - 数据库连接池
  - 日志系统
  - 缓存管理

## 三、接口设计原则

### 3.1 RESTful API设计
- 使用HTTP方法表示操作：GET、POST、PUT、DELETE
- 使用名词表示资源，避免动词
- 使用复数形式表示资源集合
- 使用HTTP状态码表示结果

### 3.2 响应格式
```json
{
  "code": 200,
  "message": "success",
  "data": {}
}
```

### 3.3 错误处理
```json
{
  "code": 400,
  "message": "参数错误",
  "data": null
}
```

### 3.4 认证方式
- 使用JWT（JSON Web Token）进行身份认证
- Token有效期：24小时
- 刷新Token有效期：7天

## 四、数据库设计

### 4.1 核心表结构

#### 用户表 (users)
```sql
CREATE TABLE users (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    username VARCHAR(50) UNIQUE NOT NULL,
    email VARCHAR(100) UNIQUE NOT NULL,
    password_hash VARCHAR(255) NOT NULL,
    role ENUM('user', 'admin') DEFAULT 'user',
    avatar_url VARCHAR(255),
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    INDEX idx_username (username),
    INDEX idx_email (email)
);
```

#### 题目表 (problems)
```sql
CREATE TABLE problems (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    title VARCHAR(255) NOT NULL,
    description TEXT NOT NULL,
    difficulty ENUM('easy', 'medium', 'hard') NOT NULL,
    time_limit INT DEFAULT 1000, -- ms
    memory_limit INT DEFAULT 128, -- MB
    input_description TEXT,
    output_description TEXT,
    sample_input TEXT,
    sample_output TEXT,
    tags JSON,
    created_by BIGINT,
    is_public BOOLEAN DEFAULT TRUE,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    FOREIGN KEY (created_by) REFERENCES users(id),
    INDEX idx_difficulty (difficulty),
    INDEX idx_created_at (created_at)
);
```

#### 提交记录表 (submissions)
```sql
CREATE TABLE submissions (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    user_id BIGINT NOT NULL,
    problem_id BIGINT NOT NULL,
    code TEXT NOT NULL,
    language VARCHAR(20) NOT NULL,
    status ENUM('pending', 'judging', 'accepted', 'wrong_answer', 
                'time_limit_exceeded', 'memory_limit_exceeded', 
                'runtime_error', 'compile_error') DEFAULT 'pending',
    runtime_ms INT,
    memory_kb INT,
    error_message TEXT,
    submitted_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (user_id) REFERENCES users(id),
    FOREIGN KEY (problem_id) REFERENCES problems(id),
    INDEX idx_user_problem (user_id, problem_id),
    INDEX idx_status (status),
    INDEX idx_submitted_at (submitted_at)
);
```

## 五、安全设计

### 5.1 代码安全
- 使用Docker容器隔离用户代码
- 限制CPU时间、内存使用、文件系统访问
- 禁止网络访问和系统调用
- 超时自动终止

### 5.2 数据安全
- 密码使用bcrypt加密存储
- 敏感数据传输使用HTTPS
- SQL注入防护：使用参数化查询
- XSS防护：输入过滤和输出编码

### 5.3 访问控制
- 基于角色的权限控制（RBAC）
- API访问频率限制
- 敏感操作需要二次确认

## 六、性能优化

### 6.1 数据库优化
- 合理设计索引
- 使用连接池
- 读写分离（可选）
- 定期数据归档

### 6.2 缓存策略
- Redis缓存热点数据
- 排行榜数据缓存
- 题目列表缓存
- 用户会话缓存

### 6.3 评测优化
- 评测任务队列化
- 多评测机并行处理
- 预编译常用代码
- 测试用例分组执行

## 七、部署架构

```yaml
      - "8080:8080"

    build: ./judge
    depends_on:
      - rabbitmq

```
